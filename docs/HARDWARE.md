# ハードウェア

未確認項目は空欄のまま配布し、現地で実測・記入する。推測値を確定値として書かない。特に報酬ポンプの吐出量、配線、電圧は動物の健康とデータ妥当性に直結するため、コードから確認できない値を埋めない。

## コード由来の確定値

Arduino TTL は [../train/pd_ttl/pd_ttl.ino](../train/pd_ttl/pd_ttl.ino) に基づきます。

| 項目 | 値 |
|---|---|
| TTL 出力ピン | `PIN = 13` |
| パルス幅 | `PULSE_MS = 100` ms |
| ボーレート | `115200` |
| プロトコル | Python が `"PULSE\n"` を送信し、Arduino が PIN を HIGH 100 ms、続けて LOW に戻す |
| 初期状態 | `digitalWrite(PIN, LOW)` |
| 受信解釈 | `Serial.readStringUntil('\n')` の後 `trim()` し、`"PULSE"` 一致時のみ発火 |
| ダブルパルス | 500 ms 後の再パルス処理はコメントアウトされており無効 |
